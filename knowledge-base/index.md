## 项目概览

`aiforge` 是一个 npm 发布的 TypeScript CLI，用来把统一知识仓库中的 Agents、Skills、Instructions 和 MCP 配置同步到不同 AI 编码工具的本地目录。当前仓库同时包含运行时代码、公开文档、真实知识样本和 brownfield 规划产物，因此它既是产品仓库，也是长期演进的项目知识基座。

## 开始使用这个知识库

### 快速进入源码上下文

先看：

1. `project-overview.md`
2. `architecture.md`
3. `component-inventory.md`

这样可以最快建立“它是什么、怎么跑、主要改哪里”的心智模型。

### 如果你要做不同类型的工作

- **新增/调整工具支持：** 先看 `architecture.md`、`component-inventory.md`，再跳到 `../docs/install-rules-matrix*.md`
- **修改安装安全逻辑：** 先看 `architecture.md` 中的安全边界章节，再看 `src/services/fs-utils.ts`、`src/stages/execute-install.ts`
- **修改 CLI 入口或输出：** 先看 `component-inventory.md` 和 `development-guide.md`
- **做 brownfield 规划/PRD：** 把本文件和 `../_bmad-output/project-context.md` 一起作为入口

## 对 AI 辅助开发的建议

- 把 `knowledge-base/index.md` 当成主入口，而不是直接从全仓库搜索开始。
- 需要运行时事实时，优先读 `src/` 和 `tests/`。
- 需要公开口径时，优先读 `README*` 与 `docs/`。
- 需要规则、历史决策和 story 证据时，优先读 `_bmad-output/`。
- 搜索全仓库前要先明确是否需要排除 `.github/skills` 和 `_bmad-output/`，否则很容易把样本/过程资产和产品源码混在一起。
