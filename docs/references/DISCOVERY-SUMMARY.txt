🎯 关键发现

【1】指令完整性
    - 主命令参数: repo-url（可选）
    - 短选项: -g, -l, -t, -d（各有长选项别名）
    - 长选项: --dry-run, --quiet, --force, --ssh, --token, --clone-dir
    - 全部 10 个选项都有清晰的默认值和说明

【2】工具支持矩阵清晰
    ┌────────────┬──────┬──────┬──────────────────┐
    │ 工具       │ 全局 │ 项目 │ 支持资源类型     │
    ├────────────┼──────┼──────┼──────────────────┤
    │ Copilot    │ ✅   │ ✅   │ 全 4 种          │
    │ Claude     │ ✅   │ ✅   │ agents, skills   │
    │ Cursor     │ ✅   │ ✅   │ agents, skills   │
    │ VS Code    │ ✅   │ —    │ mcp-tools        │
    └────────────┴──────┴──────┴──────────────────┘

【3】规则系统高度可扩展
    - 规则定义: tool + scope + sourceDir + type + targetDir + mainFile
    - 内置规则数: 16（MVP）
    - 扩展难度: 低（仅需修改 BUILTIN_RULES 数组）

【4】双重安装模式清晰
    复制模式（默认）
    ├─ 项目级和全局都支持
    ├─ 产生独立副本
    └─ 更新需重新运行 aiforge

    符号链接模式（-l）
    ├─ 仅全局安装（项目级自动降级）
    ├─ 指向持久化仓库
    ├─ git pull 自动更新
    └─ 防止团队断链接

【5】四层认证优先级实现清晰
    1. CLI 参数 (--token, --ssh)
    2. 环境变量 (AIFORGE_TOKEN, GITLAB_TOKEN, GIT_TOKEN)
    3. 配置文件 (~/.aiforge/config.json auth 字段)
    4. 系统凭据 (Keychain / 凭据管理器)

【6】用户体验设计周密
    - TTY 检测: 保证 init 命令在终端环境运行
    - 三段式错误: 什么坏了 → 为什么 → 怎么修（+ 可复制命令）
    - 预览模式: --dry-run 展示完整安装计划（不写入）
    - 冲突处理: 自动备份 (.bak) + 可选强制覆盖 (--force)
    - 零结果诊断: 结果为空时输出扫描了什么、为什么匹配失败、建议修复

【7】安全设计完整
    - Token 脱敏: glpat-ab****xy 格式显示
    - npm 包零信息: 不含任何公司域名、仓库 URL、Token
    - Manifest 跟踪: 记录文件哈希，用于冲突检测
    - 符号链接沙箱: 验证链接目标，防止路径遍历

【8】语言国际化就位
    - 支持: 中文 (zh-CN) 和英文 (en)
    - 配置位置: ~/.aiforge/config.json 的 language 字段
    - 选择时机: aiforge init 交互选择 → setLanguage() 全局设置
    - 完整性: 所有用户可见字符串都已双语化

【9】管道式架构实现
    CLI 解析 → 地址解析 → 认证 → 克隆 → 检测 → 规则匹配 → 安装 → 汇总
    7 个阶段，每个阶段清晰的输入/输出，易于测试和扩展

【10】产品定位明确
     - 核心哲学: 标准适配器，而非标准制定者
     - 工具与知识解耦: 工具发布在 npm，知识托管在私有 Git
     - 目标用户: 中高级开发者、新员工、平台维护者
     - 北极星指标: 公司开发者采用率（首月 > 50%，3 月 > 80%）
