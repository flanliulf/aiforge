基于 Anthropic 当前官方文档，若以 **Claude Code** 为基准，可以把这几类概念大致分成 6 层：**入口层（slash commands）**、**能力层（skills）**、**代理层（sub-agents）**、**自动化层（hooks）**、**外部工具层（MCP）**、**分发层（plugins）**。其中一个很关键的变化是：**custom commands 已并入 skills**；今天你看到的很多 `/xxx`，底层其实已经是 skill。 ([Claude API Docs][1])

## 我建议你这样理解它们之间的层级

最稳的理解方式是：

**Slash commands = 入口**
**Skills = 可复用能力包**
**Sub-agents = 独立执行角色**
**Hooks = 生命周期自动化 / 守卫**
**MCP = 外部工具总线**
**Plugins = 打包与分发容器** ([Claude API Docs][1])

## 两个最容易混淆的边界

**1）Skills vs Slash commands**
现在最容易误解的是把 `/xxx` 当成“概念本体”。其实 `/xxx` 更像调用入口；真正的能力实体通常是 skill。Anthropic 官方已经明确：**custom commands have been merged into skills**。也就是说，今天设计规范时，最好把 “slash command” 当成 **invoke surface**，把 “skill” 当成 **capability unit**。 ([Claude API Docs][2])

**2）Skills vs Sub-agents**
Skill 更像“给 Claude 的操作手册 / playbook”；sub-agent 则是“真正被派出去单独干活的专业角色”。一个 skill 甚至可以通过 frontmatter 指定 `context: fork`，把执行放到 subagent 上。 ([Claude API Docs][2])
